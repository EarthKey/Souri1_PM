# 「作れるのに伸びないクセ診断」結果ページ設計書

## 1. スコアリング・判定ロジック

### 基本スコア集計
6タイプそれぞれに対応する3つの質問の回答（-2〜+2点）を合計し、タイプ別基本スコア（-6〜+6点）を算出する。

- **かなり当てはまる**: +2点
- **やや当てはまる**: +1点
- **どちらとも言えない**: 0点
- **あまり当てはまらない**: -1点
- **全く当てはまらない**: -2点

### 補正の適用
- **Q19（制作圧迫時のクセ強化）**: 回答スコアが+1以上（「かなり」「やや」）の場合、最高スコアタイプの基本スコアに `+1` を加算する。
- **Q20（バイアス自覚度）**: 回答スコアが+1以上の場合、結果画面に「自覚度コメント」を追加表示するフラグをONにする。

### 主タイプ・副タイプおよび複合型の判定ルール
集計後のスコア（補正適用後）に基づき、以下の手順で最終タイプを判定する。

1. **スコア順ソート**:
   タイプ別スコアの高い順にソートし、最も高いスコアを $S_1$（主タイプ）、2番目に高いスコアを $S_2$（副タイプ）とする。
2. **複合型（ハイブリッド型）判定**:
   最高得点が **2タイプで同点**の場合、2タイプの複合型として判定する。
   - 表示例：`ペースセッター型 × 強制型`
   最高得点が **3タイプ以上で同点**の場合、**「複合傾向型」**と判定する。
   - スコア差が2点以上の場合は、単一の `主タイプ` を結果タイプとする。
3. **比率（％）表示の計算**:
   ユーザーの多面的な発信傾向を伝えるため、主タイプと副タイプのスコア比率を表示する。
   比率の算出には、負のスコアを排除するために各タイプスコア（-6〜+6）に `+6` を加算して正の値（0〜12）に変換した値（$S'_1, S'_2$）を用いる。
   - 計算式：
     - 主タイプ比率（％） = $\frac{S'_1}{S'_1 + S'_2} \times 100$ （小数点以下四捨五入）
     - 副タイプ比率（％） = $100 - \text{主タイプ比率}$
   - ※ 両方の変換後のスコア合計が0の場合（共に-6点の場合）は、一律50%/50%とする。

## 2. 結果画面の構成（表示順序：ミス先行型）

ユーザーが自分の「伸び悩みのクセ（行動）」を自覚し、そこから納得して改善アクションへ進めるように、「ミス先行型」の順序で表示する。

### 画面遷移フロー
```
最後の質問（Q20）に回答
↓
ローディングアニメーション（1.5秒程度）
「あなたの無意識のクセを分析中...」
↓
結果画面へフェードイン
```

### 結果画面のセクション構成（上から順に表示）

#### ① ⚠️ あなたが最も起こしやすい伸び悩み（最上部に配置）
診断結果が単なる性格診断ではなく「行動改善ツール」であることを示すため、最初に「伸び悩みパターン」を突きつける。
- **表示内容**:
  - 主タイプ（複合型の場合は両タイプ）の「起こしやすい伸び悩み3選」を表示。
- **デザイン**: 警告色（薄い赤または黄色背景）のカードで、視覚的なインパクトを与える。

#### ② 🔥 あなたのタイプ（および比率）
- **表示内容**:
  - **通常判定時**: 主タイプ名 ＋ 一言ラベル ＋ キャッチコピー
  - **複合判定時**: 「主タイプ名 × 副タイプ名」のハイブリッド表示
  - **比率表示**: `主タイプ割合%` と `副タイプ割合%`
- **デザイン**: タイプのテーマカラーをベースにしたグラデーションヘッダー。

#### ③ 💡 理由（あなたの傾向・ズレの説明）
なぜその伸び悩みが起きてしまうのか、発信現場での行動の「ズレ」を解説する。
- **表示内容**:
  - Type.mdの「ズレ（行動ベース）」を表示。

#### ④ 📊 あなたのバイアス傾向マップ（グラフセクション）
全体のバランス、偏り度合いを視覚的に楽しむため、「棒グラフ」と「レーダーチャート」を両方表示する。
- **棒グラフ（理解用）**:
  直感的で分かりやすいシンプルなCSS（`width` アニメーション）による横棒グラフ。
- **レーダーチャート（見栄え用）**:
  6角形のチャート。Canvas APIまたはSVGでスマートに描画。

#### ⑤ ✅ 明日からできる行動変容（OK行動 ＋ 実践セリフ）
診断を「気づき」から「行動」に落とし込むための具体的なアクションとフレーズ。
- **修正行動（1つだけ）**: Type.mdの「修正行動（1つだけ）」を表示。
- **実践セリフ**: Type.mdの「セリフ」を表示。
- **デザイン**: セリフは大きな吹き出しデザインで表示し、「クリップボードにコピー」ボタンを配置。

#### ⑥ 📌 サブ傾向の補足（単一判定の場合のみ表示）
複合判定にならなかった場合でも、2番目に高かった傾向について補助的なアドバイスを行う。

#### ⑦ 💡 バイアス自覚度コメント
Q20の判定がON（当てはまる）の場合にのみ表示されるメッセージ。
- 例：「あなたは自分のクセを自覚しにくい傾向があります。定期的に周囲（発信仲間やコミュニティの参加者）から『最近の私の投稿、どう見えてる？』とフィードバックをもらうのがおすすめです。」

#### ⑧ 📝 振り返りメモ ＆ 今日から試す行動
今日得た気づきと、明日から試す具体的な行動をその場で選んで保存できるエリア。
- **今日から試す行動（チェックリスト）**:
  - 本タイプに応じた「修正行動」が最初からチェックボックスとして表示される。
  - ユーザーが自分で自由に行動を追加できる入力欄と追加ボタン。
- **振り返りメモ**: テキスト入力欄。
- **保存仕様**: `localStorage` を使用し、ローカル環境にのみ保存される（プライバシー安全性を明記）。

#### ⑨ 💾 結果の保存・共有・ダウンロード
- **結果画像として保存**:
  - Canvas API を用いて、結果画面の「ミス3選」「タイプ名・比率」「グラフ」を統合した美しい診断結果画像を生成し、PNGとしてダウンロードできるボタン。
- **結果PDF保存 / 印刷用レイアウト**:
  - `window.print()` を起動するボタン。A4用紙1枚にきれいに収まるスタイルシート（CSS `@media print`）を適用する。
- **SNSで改善を宣言（共有導線）**:
  - 恥ずかしいタイプ名を晒すのではなく、「私のミスのクセ対策宣言」として投稿できるテンプレートを生成。
  - テキスト例：「『作れるのに伸びないクセ診断』を実行しました！明日から【1投稿出したら、最低30分は次へ進まず反応を観察する】を徹底します！ #作れるのに伸びないクセ診断 [URL]」
- **「もう一度診断する」ボタン**。

## 3. 保存・印刷用レイアウトの設計（CSS `@media print`）

印刷またはPDF保存時に、以下のスタイルを自動適用する。
- 画面上の「回答ボタン」「保存ボタン」「もう一度診断する」などの操作系UIを完全に非表示にする（`display: none`）。
- 振り返りメモ欄の入力済みテキストや、チェックボックスの選択状態がそのまま印刷紙面に反映されるようにする。
- ページの背景色を白、文字色を黒にし、プリンターのインク消費を抑えつつ視認性を高める。

// 判定ロジック関数
function analyzeResult(answers) {
  // answersは 0〜19 のインデックスを持つ長さ20の配列 (値: -2 〜 +2)
  
  // 各タイプのスコア初期化
  const scores = {
    pacesetter: answers[0] + answers[1] + answers[2],
    coach: answers[3] + answers[4] + answers[5],
    authoritative: answers[6] + answers[7] + answers[8],  // 強制
    affiliative: answers[9] + answers[10] + answers[11],  // 関係重視
    visionary: answers[12] + answers[13] + answers[14],  // 先見
    democratic: answers[15] + answers[16] + answers[17],   // 民主
  };
  
  // 補正Q19の適用（制作圧迫時のクセ強化）
  const isQ19Active = answers[18] >= 1;
  let sortedTypes = Object.keys(scores).sort((a, b) => scores[b] - scores[a]);
  let maxScore = scores[sortedTypes[0]];
  let topScoreCount = sortedTypes.filter(type => scores[type] === maxScore).length;
  if (isQ19Active && topScoreCount === 1) {
    scores[sortedTypes[0]] += 1;
  }
  
  // 補正Q20の適用（自覚度フラグ）
  const isAwarenessLow = answers[19] >= 1;
  
  // スコア順にソート
  sortedTypes = Object.keys(scores).sort((a, b) => scores[b] - scores[a]);
  const primaryType = sortedTypes[0];
  const secondaryType = sortedTypes[1];
  const tertiaryType = sortedTypes[2];
  
  const primaryScore = scores[primaryType];
  const secondaryScore = scores[secondaryType];
  maxScore = primaryScore;
  topScoreCount = sortedTypes.filter(type => scores[type] === maxScore).length;
  
  // 複合型判定：最高得点が複数タイプで同点
  const isHybrid = topScoreCount === 2;
  const isComposite = topScoreCount >= 3;
  
  // 比率計算（スコアに+6して正の値に変換）
  const pScoreNorm = primaryScore + 6;
  const sScoreNorm = secondaryScore + 6;
  const totalNorm = pScoreNorm + sScoreNorm;
  
  let primaryRatio = 50;
  let secondaryRatio = 50;
  if (totalNorm > 0) {
    primaryRatio = Math.round((pScoreNorm / totalNorm) * 100);
    secondaryRatio = 100 - primaryRatio;
  }
  
  return {
    primaryType,
    secondaryType,
    tertiaryType,
    isHybrid,
    isComposite,
    primaryRatio,
    secondaryRatio,
    scores,
    isAwarenessLow
  };
}
