# 「作れるのに伸びないクセ診断」配色デザイン案

## 結論

推奨は **案1「手描きの改善ノート」**。

完成した診断イラストは、色鉛筆・マーカー・紙の質感が強い。現在のHBR風デザインは信頼感がある一方、無機質で、イラストだけが別作品に見えやすい。案1なら、診断画面から結果イラストまでを一つの体験としてつなげられる。

---

## 案1：手描きの改善ノート

### コンセプト

**「自分のクセを書き込み、明日の行動を一つ決めるワークノート」**

白ではなく温かい紙色を基調にし、深い緑を信頼色、朱赤を注意色、黄土色を前向きなアクセントとして使う。完成イラストの色鉛筆感と最も自然につながる。

### カラーパレット

| 役割 | 色 | HEX |
|---|---|---|
| ページ背景 | 生成り紙 | `#F3EEDF` |
| カード背景 | 明るい紙白 | `#FFFDF7` |
| メインカラー | 深い苔緑 | `#356859` |
| メイン濃色 | 墨緑 | `#244A40` |
| アクセント | 鉛筆の黄土色 | `#D9A441` |
| NG・警告 | 朱赤 | `#D95645` |
| OK・改善 | 若葉緑 | `#4E8A57` |
| 本文 | 墨色 | `#242521` |
| 補助文字 | 灰茶 | `#68665F` |
| 罫線 | 薄い鉛筆線 | `#D8D0BE` |

### 色の使い方

- 背景70%、紙白20%、深い苔緑7%、黄土色・朱赤3%程度に抑える
- 開始ボタンは深い苔緑、選択中は黄土色の下線や枠で示す
- 診断結果の「伸びないクセ」は朱赤、「明日の1アクション」は若葉緑で明確に分ける
- カードにごく薄い紙目・罫線表現を加える
- 6タイプ固有色はグラフと小さなラベルだけに使用し、画面全体をカラフルにしすぎない

### 長所

- 完成イラストと最も相性がよい
- 「診断」よりも「改善ワーク」というサイト本来の目的が伝わる
- 温かいが幼くならず、コミュニティ企画にも合う
- 現在のHTML構造を大きく変えずに実装できる

### 弱点

- 紙の質感を強くしすぎると、古臭く見える
- 朱赤を多用すると注意喚起ばかりの画面になるため、使用量の制御が必要

## 案2：静かな編集室

### コンセプト

**「感情ではなく、自分の発信を冷静に編集する診断」**

現在のHBR風デザインを磨き直す案。青緑とアイボリーを中心に、オレンジを一点だけ効かせる。信頼感と読みやすさを最優先する。

### カラーパレット

| 役割 | 色 | HEX |
|---|---|---|
| ページ背景 | 淡い灰緑 | `#EEF2EF` |
| カード背景 | 編集紙白 | `#FCFDFB` |
| メインカラー | 深い青緑 | `#275E5A` |
| メイン濃色 | 濃紺緑 | `#183D3A` |
| アクセント | 焦がしオレンジ | `#D78335` |
| NG・警告 | 深い赤 | `#C94B45` |
| OK・改善 | 落ち着いた緑 | `#3F7D5A` |
| 本文 | 青みの墨色 | `#202A2A` |
| 補助文字 | 石板色 | `#66706D` |
| 罫線 | 淡い青灰 | `#D5DEDA` |

### 色の使い方

- 青緑を見出し・ボタン・進捗バーに統一して使う
- オレンジは重要な数値、選択中、結果の核心だけに使う
- タイプ色はグラフ内に限定し、結果画面の情報密度を整理する
- イラストの周囲に紙白の余白を広く取り、展示物のように見せる

### 長所

- 信頼性、読みやすさ、診断ツールらしさが最も高い
- 現在のデザイン資産を活用しやすく、改修コストが低い
- 長文の診断結果を落ち着いて読ませやすい

### 弱点

- 安全だが、作品としての記憶には残りにくい
- 完成イラストの手描き感との温度差は残る
- 現状からの変化が小さく、コンテスト作品としては弱い

## 案3：夜のバイアス研究所

### コンセプト

**「見えなかった発信バイアスを、暗室で分析・可視化する研究所」**

深い紺色を背景に、ミントグリーンと琥珀色を発光アクセントとして使う。診断中は分析装置のように見せ、結果画面だけ明るい紙色へ切り替える。

### カラーパレット

| 役割 | 色 | HEX |
|---|---|---|
| ページ背景 | 夜紺 | `#101B26` |
| 診断カード | 濃い青灰 | `#172733` |
| 結果カード | 明るい紙白 | `#FFFDF7` |
| メインカラー | ミント光 | `#67C7B5` |
| アクセント | 琥珀色 | `#F2B84B` |
| NG・警告 | 珊瑚赤 | `#F06A61` |
| OK・改善 | 明るい緑 | `#63C174` |
| 暗背景の本文 | 薄い青白 | `#E8F0EF` |
| 明背景の本文 | 墨色 | `#252A2B` |
| 暗背景の罫線 | 青灰 | `#304654` |

### 色の使い方

- 導入・質問画面は暗色、結果画面は紙白へ切り替える
- 進捗バーや選択状態をミント光で示す
- 結果表示時に背景が明るくなり、「分析から行動へ移る」変化を演出する
- タイプ固有色は暗背景上で発光ラベルとして使う

### 長所

- 3案の中で最も印象に残る
- 診断中と結果後の体験変化を色で表現できる
- AI・バイブコーディング文脈との相性がよい

### 弱点

- 完成した柔らかい手描きイラストとは最も相性が悪い
- 暗色画面は長文を読む診断サイトでは疲れやすい
- 実装とアクセシビリティ調整の工数が大きい
- 見た目の派手さが、Boring戦略の思想より前に出る危険がある

## 推奨判断

**案1「手描きの改善ノート」を採用する。**

理由は、単に色がかわいいからではない。サイトのゴールは「タイプを当てること」ではなく、利用者が自分のクセを書き留め、明日の行動を一つ変えることにある。ノートという視覚言語は、その目的と一致する。

案2は堅実だが、現状維持に近い。案3は目立つが、企画の思想とイラストを犠牲にしている。コンテストで見栄えを狙って案3へ寄せるのは、サイトの本質から外れるため推奨しない。
